POSLink Full-Integration Guide

The library provides a wide range of classes for communication between ECR and POS.

This document provides an overview of the important categories of features available in the library, and specific classes you should know about when building your app.

The whole library contains 7 dlls. (Support .Net Framework 4.6.2 or newer)

  • netstandard is the .NET Standard 2.0 library.
  • POSLinkCore provides the Log class and communication classes.
  • POSLinkAdmin provides some core common command classes.
  • POSLinkFullIntegration provides FullIntegration-only classes.
  • Renci.SshNet is used to do UploadLog which using SFTP.
  • POSLinkUart provides a way to do communication with UART, for the .NET Standard 2.0 is not support SerialPort. The demo about it is in Samples\POSLinkUart.
  • SshNet.Security.Cryptography Renci.SshNet depends on this.

Libs folder: contain dlls that will be used in development. ComRegister folder: if you need to register COM component, please refer to it. It also contains tlb files in it that are used to develop through COM component.

Doc folder: API Guide folder: contain the API guide about dlls. Reference folder: this document is aimed to provide the information that may be used in programming. This is in addition to the API guide. It can be consulted when the relevant information cannot be found in the API guide.

Samples folder: FultterDemo folder: Code folder: contains flutter demo code. Demo folder: contains the POSLink flutter demo. POSLinkUart folder: contains the UART communication dll sample. .NetFrameworkDemo folder: contains a simple .Net Framework POSLink demo. MAUI folder: GlCommBinding folder: contains a UART sample for Android in MAUI demo. POSLinkMauiDemo folder: contains the POSLink MAUI demo.

Full Integration Architecture

Initialization

Add dll to reference at first.

Set log setting if needed

xxxxxxxxxx
POSLinkFullIntegration.POSLinkFull poslink = POSLinkFullIntegration.GetPOSLinkFull();
POSLinkCore.LogSetting logSetting = new POSLinkCore.LogSetting();
logSetting.Enable = true;
logSetting.Level = POSLinkCore.LogSetting.LogLevel.Debug;
logSetting.Days = 30;
logSetting.FilePath = ".\\";
poslink.SetLogSetting(logSetting);

Set communication setting.

xxxxxxxxxx
POSLinkCore.CommSetting.TcpSetting tcpSetting = new POSLinkCore.CommSetting.TcpSetting();
tcpSetting.Ip = "127.0.0.1";
tcpSetting.Port = 10009;
tcpSetting.Timeout = 60000;
Note: Since .net standard doesn't have the SerialPort library, Inherit and override the CustomerCommSetting class. Please see the sample of POSLinkUart.

Then run the command you want.

xxxxxxxxxx
POSLinkFullIntegration.Terminal terminal = poslink.GetTerminal(tcpSetting);
POSLinkAdmin.Manage.InitRsp doInitRsp = null;
POSLinkAdmin.ExecutionResult executionResult = null;
executionResult = terminal.Manage.Init(out doInitRsp);
if(executionResult.GetErrorCode() == POSLinkAdmin.ExecutionResult.Code.Ok)
{
    Console.WriteLine(doInitRsp.ResponseCode);
}

COM component invocation

C# can directly use the POSLink SDK. However, not all programming languages have this capability. For other programming languages, the Windows operating system provides the COM component technology, which can be used by all programming languages to access POSLink. Here's how to use POSLink through the COM component technology, which can be used by languages such as C++, Delphi, and so on. Here is a way to register the COM component and to develop:

  1. Environment configuration. Using regasm.exe to register COM component. Ensure you have installed .NET Framework 4.0 or later. The .NET Framework download address is: https://dotnet.microsoft.com/en-us/download/dotnet-framework and regasm.exe instruction manual is: https://learn.microsoft.com/en-us/dotnet/framework/tools/regasm-exe-assembly-registration-tool.

    Please ensure regasm.exe in %SYSTEMROOT%\Microsoft.NET\Framework64\v4.0.30319 or %SYSTEMROOT%\Microsoft.NET\Framework\v4.0.30319. If your regasm.exe in another path, please edit the regasm.exe path in Libs\ComRegister\x64\RegisterCOMx64_Net4.0.bat and Libs\ComRegister\x86\RegisterCOMx86_Net4.0.bat.

  2. Register the COM component. If you are developing a program that is 64-bit, please use Libs\ComRegister\x64\RegisterCOMx64_Net4.0.bat to Register the COM component. If you are developing a program that is 32-bit, please use Libs\ComRegister\x86\RegisterCOMx86_Net4.0.bat to Register the COM component.

  3. Development. Each framework uses its own way to invoke COM components. For example: C++ and Delphi use POSLinkCore.tlb, POSLinkAdmin.tlb, POSLinkSemiIntegration.tlb and POSLinkUart.tlb to develop, Java use ActiveXComponent to develop, VB use POSLinkSemiIntegration object.

For more information about COM components, please see: https://learn.microsoft.com/en-us/windows/win32/com/guide.

Attention

When developing Android application using Xamarin or MAUI, you may encounter SSL Authentication failed exception. This issue is addressed in https://github.com/xamarin/xamarin-android/pull/6665. You need to disable server certificate validation on Android to resolve. Example for MAUI: Add this in MauiProgram.cs.

xxxxxxxxxx
#if ANDROID && DEBUG
    Platforms.Android.DangerousTrustProvider.Register();
#endif

Create the DangerousTrustProvider class.

xxxxxxxxxx
using Java.Security;
using Java.Net;
using Javax.Net.Ssl;
using Java.Security.Cert;


namespace POSLinkMauiDemo.Platforms.Android
{
    internal class DangerousTrustProvider : Provider
    {
        private const string TRUST_PROVIDER_ALG = "DangerousTrustAlgorithm";
        private const string TRUST_PROVIDER_ID = "DangerousTrustProvider";


        public DangerousTrustProvider() : base(TRUST_PROVIDER_ID, 1, string.Empty)
        {
            var key = "TrustManagerFactory." + DangerousTrustManagerFactory.GetAlgorithm();
            var val = Java.Lang.Class.FromType(typeof(DangerousTrustManagerFactory)).Name;
            Put(key, val);
        }
    
        public static void Register()
        {
            Provider registered = Security.GetProvider(TRUST_PROVIDER_ID);
            if (null == registered)
            {
                Security.InsertProviderAt(new DangerousTrustProvider(), 1);
                Security.SetProperty("ssl.TrustManagerFactory.algorithm", TRUST_PROVIDER_ALG);
            }
        }
    
        public class DangerousTrustManager : X509ExtendedTrustManager
        {
            public override void CheckClientTrusted(X509Certificate[] chain, string authType, Socket socket) { }
            public override void CheckClientTrusted(X509Certificate[] chain, string authType, SSLEngine engine) { }
            public override void CheckClientTrusted(X509Certificate[] chain, string authType) { }
            public override void CheckServerTrusted(X509Certificate[] chain, string authType, Socket socket) { }
            public override void CheckServerTrusted(X509Certificate[] chain, string authType, SSLEngine engine) { }
            public override void CheckServerTrusted(X509Certificate[] chain, string authType) { }
            public override X509Certificate[] GetAcceptedIssuers() => Array.Empty<X509Certificate>();
        }
    
        public class DangerousTrustManagerFactory : TrustManagerFactorySpi
        {
            protected override void EngineInit(IManagerFactoryParameters mgrparams) { }
            protected override void EngineInit(KeyStore keystore) { }
            protected override ITrustManager[] EngineGetTrustManagers() => new ITrustManager[] { new DangerousTrustManager() };
            public static string GetAlgorithm() => TRUST_PROVIDER_ALG;
        }
    }
}
