Encrypted Sensitive TLV Data Format Definition

The follow is the illustration for format.
cn - Compressed numeric data elements consist of two numeric digits (having values in the range Hex '0'–'9') per byte. These data elements are left justified and padded with trailing hexadecimal 'F's. Example: The Application Primary Account Number (PAN) is defined as "cn" with a length of up to ten bytes. A value of 1234567890123 may be stored in the Application PAN as Hex '12 34 56 78 90 12 3F FF' with a length of 8.
b - These data elements consist of either unsigned binary numbers or bit combinations that are defined elsewhere in the specification. Binary example: The Application Transaction Counter (ATC) is defined as “b” with a length of two bytes. An ATC value of 19 is stored as Hex '00 13'.

Notes:

  1. "\C" "\L" "\R" will not change line automatically, please add "\n" if you want to change to next line.
  2. "\B" can be used along with "\L" or "\R" or "\C" with format like "\B\R".
  3. "\B" will automatically jump to next line if there are some words before "\B".
  4. "\B" will only bold the first line.
  5. "\1", "\2", and "\3" will only apply the font size from where the control character was started and to the end of that specific lines text.
