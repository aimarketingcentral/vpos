◆ CustomData

string [] POSLinkFullIntegration.FullIntegration.InputAccountWithEmvRequest.CustomData
getset

Custom data string to be encrypted, used to define any special formats for sensitive data.

Custom data must be in hexadecimal format. Placeholder strings should not be converted to hex.

Placeholder format:

${ PARAMETER_NAME }

${ VARIABLE_NAME, FORMAT, PADLEN}

PARAMETER_NAME - Parameter name, optional values:TRACK1DATA, TRACK2DATA, TRACK3DATA.

VARIABLE_NAME - Variable name, optional values:TRACK1LEN, TRACK2LEN, TRACK3LEN.

FORMAT - The format of track data length, optional values:HEX.

PADLEN - The number of padding bytes.

Example: Original string = "pq${TRACK2DATA}abc"

Convert custom string to hex = "7071${TRACK2DATA}616263"

◆ EncryptionFlag

Const.DataEncryptionFlag POSLinkFullIntegration.FullIntegration.InputAccountWithEmvRequest.EncryptionFlag
getset

Data Encryption flag.

Default is NotEncrypted. In BroadPOS P2PE, this field is mandatory and can not set to NotEncrypted.

The application will convert it from hexadecimal binary array into an ASCII string. For example, the 3 bytes binary array 0x1a 0x23 0xd4 can be converted into "1a23d4".

◆ TagList

string POSLinkFullIntegration.FullIntegration.InputAccountWithEmvRequest.TagList
getset

EMV tag list to be returned in EMV TLV data in response message.

Please see EMV Tags for Response of Authorize Card in Reference.

Example: Request for tag 0x9F26 0x9F27 0x4F, then this field would be: "9F269F274F".

Cannot request the following sensitive tags:

0x56: track 1 data

0x57: track 2 equivalent data

0x5A: PAN

0x9F1F: track1 discretionary data

0x9F6B: track 2 data(MasterCard Only)

0x9F20: track 2 discretionary data

Attribute: an...1024
