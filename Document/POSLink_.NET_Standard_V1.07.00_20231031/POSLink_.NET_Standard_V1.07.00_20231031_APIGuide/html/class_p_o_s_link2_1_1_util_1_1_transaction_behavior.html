◆ AcceptedCardType

string POSLink2.Util.TransactionBehavior.AcceptedCardType
getset

The card type bitmap.

It makes POS system available to control which card types accepted for current command. and it is mandatory for fleet transaction. Please see Card Type Bitmap Definition in POSLink-Reference.

Attribute: n32

◆ AccessibilityPinPad

string POSLink2.Util.TransactionBehavior.AccessibilityPinPad
getset

The standard of accessibility to support and the status and visibility of "Accessibility Switch" on the Input Account screen.

0: Do not support A11Y pin pad; No switch.

1: On(A11Y)-Display "Accessibility Switch" in "ON" position; Use A11Y pin pad.

2: Off(A11Y)-Display "Accessibility Switch" in "OFF" position; Do not use A11Y pin pad.

Empty Value/Null: Follow TMS settings for "Accessibility Switch" visibility and value.

Attribute: n1

◆ CardPresentMode

string POSLink2.Util.TransactionBehavior.CardPresentMode
getset

Card Present mode.

2: Don't prompt, assume card not present.

3: Prompt for use to select.

◆ CofInitiator

string POSLink2.Util.TransactionBehavior.CofInitiator
getset

Indicates if the credential/card on file (CoF) transaction is customer initiated or merchant initiated. Utilized for subsequent CoF transactions (a token is used to process the transaction), otherwise it will be ignored for other transactions.

C: Customer

M: Merchant

Attribute: a1
