Encrypted Sensitive TLV Data Format Definition

The follow is the illustration for format. cn - Compressed numeric data elements consist of two numeric digits (having values in the range Hex '0'–'9') per byte. These data elements are left justified and padded with trailing hexadecimal 'F's. Example: The Application Primary Account Number (PAN) is defined as “cn” with a length of up to ten bytes. A value of 1234567890123 may be stored in the Application PAN as Hex '12 34 56 78 90 12 3F FF' with a length of 8 b - These data elements consist of either unsigned binary numbers or bit combinations that are defined elsewhere in the specification. Binary example: The Application Transaction Counter (ATC) is defined as “b” with a length of two bytes. An ATC value of 19 is stored as Hex '00 13'.

Heartland Portico Specifics

ECR Transaction ID

Terminal will map it to the <ClientTxnId> field in transaction request message. The <ClientTxnId> can only accept numeric value which means the current "ECR reference number" cannot not be used. Can be as long as 50 digits.

Adjustment

Heartland Portico Gateway only support transaction "edit", so POS needs to send total amount (base amount + tip amount) to the terminal when using HREF to do Tip Adjustment, since the terminal doesn’t know the original sale amount when the adjustment is taken place on a different terminal, it will just send the amount whatever the POS system sends.
When the adjust is done on the same terminal and use OrigRefNum, the POS just needs to send the tip amount to the terminal to do Tip Adjustment.

TransIT Specifics

Transaction Adjustment

Transaction adjustment supports four types of amounts: base transaction amount, tip amount, surcharge fee, and tax amount.
Transaction amount in the ECR-Terminal request message is base transaction amount + tax amount. So, the total amount when all values are set will be transaction amount+tip amount+surcharge fee. BroadPOS TransIT app supports transaction adjustment through local reference numer and host reference number.
Using Local Reference Number:
If user doesn't pass tip amount, tax amount, and surcharge fee, the BroadPOS app will set the missing values with original transactions corresponding amounts.
Using Host Reference Number:
If user doesn't pass tip amount, tax amount, and surcharge fee, the BroadPOS app will set these values to "0" in the host request message.
Note: Transaction adjustment API is only supported by BroadPOS TransIT app..
